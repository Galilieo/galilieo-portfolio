# 归档航线日志与整卡反馈设计规格

## 目标

把归档页从松散的左右卡片串改成结构明确的“航线日志”，并在归档末尾接入昼夜纸舟场景。同时统一站内真正可点击卡片的悬停、键盘聚焦和触控反馈，让交互边界清晰但不过度动画化。

用户应记住的核心画面是：文章和个人阶段精确落在同一条航道上，时间轴最终自然化成水纹与纸舟。

## 范围

本次包含：

- 归档页博客笔记和个人轨迹的时间线布局、层级与整卡反馈。
- 归档末尾的无卡片昼夜纸舟场景、延长过渡和分区透明度。
- 首页“主要项目 / 精选文章”、项目列表卡片、博客列表卡片的可点击反馈统一。
- 桌面、平板、移动端、浅色、深色、Reduced Motion、无 JavaScript 和键盘操作验证。

本次不包含：

- 不改变文章、项目和个人轨迹的数据来源、排序或路由。
- 不给音乐、GitHub、About 等普通信息面板添加误导性的整卡点击反馈。
- 不引入新框架、图标库、动画运行时或第三方图片。
- 不调整站点整体导航、首页信息层级、心屿主体素材和文章正文布局。

## 视觉方向

整体采用“航线日志”式编辑设计，延续当前低饱和蓝紫夜景、浅色雾蓝和克制玻璃表面。个人轨迹继续使用现有青绿色语义色，不增加新的随机强调色。

个性来自时间线的章节、航道和信标关系，而不是散落的小图标。月份标记像航程章节，中央节点像定位信标，左右卡片像交替记录的日志。纸舟只在页尾形成一次视觉记忆点，不成为第二个首页 Hero。

## 归档时间线

### 桌面骨架

- 时间线内容最大宽度沿用 `1060px`。
- 每条记录使用固定三列骨架：`minmax(0, 1fr) 92px minmax(0, 1fr)`。
- 奇数记录放左列，偶数记录放右列；中间 `92px` 只负责轴线、节点和连接线。
- 左右卡片宽度完全对称，不使用分别计算的负偏移定位。
- 节点中心与卡片顶部日期元信息行中心对齐，容许的视觉误差不超过 `1px`。
- 连接线从航道中心延伸到卡片边缘，左右长度一致。

### 月份章节

- 每个月份在三列骨架上方形成一个章节行。
- 章节标记占据中央 `92px` 航道，显示 `YYYY.MM` 和短英文辅助说明。
- 左右使用逐渐淡出的细线连接章节标记，不使用独立大胶囊。
- 多个月份之间保留明确段落间距，但中央轴保持连续。

### 卡片层级

卡片继续使用完整 `<a>`，内部固定为四层：

1. 日期与类别 / 阶段。
2. 标题。
3. 最多两行描述。
4. 标签和阅读目标。

桌面卡片建议最小高度 `232px`，使左右节奏一致；移动端取消固定最小高度。`order` 继续只用于内部排序，不显示为卡片编号。

博客卡片使用现有蓝紫强调色。个人轨迹沿用青绿色，并显示已有的阶段字段和目标链接。内容、标签数量和目标文案仍由当前 Astro 数据生成，不在 CSS 或客户端脚本中复制。

### 卡片反馈

鼠标悬停或键盘聚焦整卡时，同一条记录同步发生：

- 卡片上移最多 `3px`，边框和顶部细光增强。
- 连接线由低对比变为强调色。
- 信标中心填充，外环只扩张一次。
- 阅读箭头向前移动约 `4px`。

默认状态不持续闪烁。触控设备不依赖 hover 才能理解链接；浏览器原生点击和路由行为保持不变。

### 移动端

`760px` 及以下切换为左轴：

- 航道位于页面内容左侧约 `34px`。
- 卡片从航道右侧开始，节点和日期行仍对齐。
- 所有记录单列排列，不保留左右交替的空白。
- 月份章节缩成左侧航标加向右渐隐的细线。
- `580px` 以下沿用项目现有的无 backdrop-filter 降级。

## 视图切换

保留现有博客笔记 / 个人轨迹 tab 语义、左右方向键、Home / End 键、`?view=personal` 深链接和无 JavaScript 双面板回退。

视觉上只调整激活背景、计数层级和间距。切换时不新增依赖 JavaScript 的内容渲染，也不改变 `archive-timeline.ts` 的清理职责。

## 纸舟收尾

### 素材

使用已确认的两张原创现代数字动画素材：

- 日间：浅青天空、白云、清透海面和珍珠白纸舟。
- 夜间：深蓝天空、蓝紫月光、冷色海面和通透纸舟。

两张图片保持相同构图、海平线、纸舟位置和比例。素材分别保存为 `src/assets/images/archive/archive-voyage-day.png` 和 `src/assets/images/archive/archive-voyage-night.png`，由 Astro 图片管线生成优化格式；源码不引用临时目录或视觉原型文件。

### 布局

- 场景无卡片外框、圆角、描边和独立阴影。
- 桌面高度保持 `270px–370px`，移动端约 `360px`。
- 图片保持已确认的完整构图，不向上侵入时间线，也不使用造成明显裁切的顶部重叠方案。
- 左侧文案位于原图留白区，右侧保留纸舟完整轮廓。
- 图片四周通过页面底色和蒙版淡出，不出现生硬矩形边界。

### 过渡与透明度

- 桌面过渡长度约 `232px`，移动端约 `196px`。
- 中轴逐渐变细，经少量微光点扩散为低对比水纹，再进入海景。
- 夜间海景基础透明度约 `76%`，日间约 `68%`。
- 纸舟附近单独补回约 `14%–16%` 清晰度，使背景退后但主体不发灰。

### 动效

- 场景只允许极慢的镜头漂移、一次短水面流光和克制微光点。
- 不拆分或重画纸舟，不使用 JavaScript 重绘 SVG。
- `prefers-reduced-motion: reduce` 下停止漂移、流光、节点扩张和入场位移。
- 无 JavaScript 时昼夜素材仍由当前主题属性正确显示，文案和链接保持可读。

如果图片加载失败，页面底色、过渡线和文案仍构成完整收尾，不影响归档内容或导航。

## 站内整卡反馈统一

只对已有完整链接语义的区域增强反馈：

- 首页主要项目。
- 首页精选文章。
- 项目列表卡片。
- 博客列表卡片。
- 归档时间线卡片。

统一规则为：边框增强、最多 `2px–3px` 上移、内部媒体轻微缩放或明度调整、目标箭头小幅位移、清晰 `:focus-visible`。同一组件内部只保留一个主要运动层，避免外层和内层同时大幅移动。

音乐、GitHub 活动、About、统计和其他没有整卡链接的面板保持信息面板反馈，不伪装成可点击卡片。

## 实现边界

- `src/pages/archive/index.astro` 继续负责内容集合分组和 tab 面板结构。
- 新建 `src/components/archive/ArchiveVoyage.astro`，只负责昼夜素材、装饰性过渡和可访问性标记。
- `src/styles/interior.css` 负责时间线、项目 / 博客列表卡片和纸舟收尾样式。
- `src/styles/home/sections.css` 负责首页两个精选区域的整卡反馈。
- `src/scripts/archive-timeline.ts` 只在确有必要时做最小调整；主要内容和动效不依赖客户端渲染。
- 所有颜色优先复用 `src/styles/tokens.css` 的语义变量；需要新增时同时定义浅色和深色值。
- 不修改 `dist/`、`.astro/`、`node_modules/` 或 `docs/assets/` 参考图。

## 可访问性与渐进增强

- 保留完整锚点、tablist、tabpanel、时间元素和标题层级。
- 装饰节点、航线、水纹与图片使用 `aria-hidden="true"` 或空 `alt`。
- 键盘聚焦必须获得与 hover 等价的边框、航线、信标和箭头反馈。
- 不依赖颜色单独表达链接、激活视图或内容类别。
- 无 JavaScript 时两个归档面板顺序展示，tab 控件隐藏，纸舟收尾只出现一次。
- 页面不得产生横向滚动，触控目标保持至少现有尺寸。

## 验收

自动验证：

```bash
node scripts/verify.mjs
```

浏览器验证至少覆盖：

- `1440×1000`、`1024×768`、`390×844`。
- 浅色和深色主题。
- 博客笔记和个人轨迹切换及 URL 状态。
- 鼠标悬停、Tab 聚焦、方向键切换和触控布局。
- Reduced Motion、禁用 JavaScript、Astro 页面切换。
- 控制台错误、图片失败回退和横向溢出。

完成标准：桌面左右卡片等宽；节点与日期行对齐；移动端航道位于卡片左侧；纸舟图片无突兀边缘和明显裁切；可点击卡片有一致反馈，普通信息面板不产生误导反馈；完整验证命令通过。
