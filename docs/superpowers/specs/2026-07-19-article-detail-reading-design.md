## 目标

把现有文章详情从单一阅读面板整理成更清楚的“封面、文章身份、正文、阅读辅助、延伸阅读”结构，同时修复长文章在特定视口中可能因整块 Reveal 无法触发而保持隐藏的问题。

完成后的页面应当：

- 一眼分清封面、返回入口、标题、元信息和正文起点。
- 桌面端利用右侧空余空间提供 Recommended 和本文目录，不重复作者卡与音乐卡。
- 移动端保持正文优先，把 Recommended 降级到页尾，不占首屏也不插入正文。
- 无 JavaScript、Reduced Motion、长文章和短视口下都能直接阅读完整正文。
- 延续当前雾蓝双主题、克制玻璃质感和编辑式排版，不变成对参考站的视觉复刻。

## 页面结构

### 文章顶部

文章详情主面板顶部依次为：

1. 文章封面。
2. 返回博客入口。
3. 标题、发布日期、Category、Tags 等现有真实元信息。
4. 平板和移动端的折叠本文目录。
5. 正文。

封面与标题采用分离布局：封面作为主面板顶部的独立媒体区域，标题和元信息放在封面下方的阅读区，不把主要文字压在图片上。这样可避免不同封面明暗导致对比度不稳定，也让无正式封面的回退图仍能保持一致结构。

封面继续复用项目现有的 `getBlogCover` 选择规则：优先文章 frontmatter 的正式封面，否则使用现有稳定回退图。不得引入参考站图片、远程热链或新的第三方素材。

返回入口指向博客目录，而不是只调用浏览器历史返回，确保从外部链接直接进入文章时仍有稳定的上一级导航。

### 正文

正文继续使用当前舒适阅读宽度，保留 Markdown/MDX 已有的标题、代码块、表格、引用、图片和链接样式。

不得继续把整篇 `.prose` 作为一个需要达到可见阈值的 Reveal 目标。正文必须在初始 HTML 中可见；如保留入场反馈，只能作用于尺寸有限的文章头部或独立小区块，并且不能成为正文可见性的前置条件。

### 桌面右栏

宽桌面右栏只包含两个区块：

1. `Recommended`：最多 3 篇延伸阅读。
2. `本文目录`：由当前文章 H2/H3 生成，保持吸顶。

不增加作者卡、音乐卡、社交卡或评论区。作者信息已有 About 页面，全站音乐已有独立播放器；重复它们只会挤压阅读辅助信息。

Recommended 位于目录上方。目录承担长文定位，因此在用户滚动离开推荐区后仍保持吸顶；推荐列表本身不固定，不与目录争夺可视高度。

### 平板与移动端

在现有文章双栏布局收敛为单栏时，桌面右栏整体退出布局：

- 本文目录变成标题元信息下方的原生折叠入口，默认收起。
- 正文中间不插入 Recommended。
- 正文结束后先显示上一篇 / 下一篇，再显示最多 2 条紧凑 Recommended。
- 移动端推荐使用文字列表或低密度小卡，不重复大封面，不造成第二个文章目录页。

移动端最终阅读顺序为：

`封面 → 返回博客 → 标题与元信息 → 折叠目录 → 正文 → 上一篇/下一篇 → Recommended`

## Recommended 规则

推荐数据在 Astro 构建阶段生成，不依赖浏览器请求、数据库或客户端渲染。

对当前文章之外的公开文章依次计算：

1. 同 Category 的文章优先。
2. 在同一优先级内，共享 Tags 数量更多的文章优先。
3. 仍相同时，按 `publishedAt` 从新到旧排序。
4. 桌面端取前 3 篇，移动端显示同一结果的前 2 篇。

如果同 Category 或共享 Tags 的文章不足，允许用最新公开文章补齐。草稿、当前文章本身和无效内容不得进入结果。该规则需要写成可单元测试的纯函数，避免推荐顺序散落在 Astro 模板中。

推荐项至少显示标题和一项轻量辅助信息，可选 Category、发布日期或阅读时长中的现有真实字段；不得虚构热度、浏览量或“热门”排名。

## 视觉方向

文章详情继续使用现有内页背景和主题遮罩。主面板是比首页卡片更实、更适合长时间阅读的玻璃表面：

- 封面保持明确宽高比和主面板上沿圆角，避免裁切成全屏背景。
- 阅读区、右栏和推荐卡复用现有语义 token，同时维护浅色与深色值。
- 桌面可以使用现有克制的 `backdrop-filter`；移动端继续降低或关闭大面积模糊，并通过更实的背景色保证文字对比度。
- 不复制参考站的粉色配色、精确圆角、阴影、字体组合、角色素材或悬浮组件。
- Hover 只使用小幅边框、颜色或不超过 `2px` 的位移变化；触控设备不依赖 Hover 表达可点击性。

## 响应式与可访问性

- 宽桌面显示正文与右栏；在现有约 `1100px` 的文章布局断点附近切换为单栏，不为推荐列表新增横向滚动。
- 目录锚点保持真实 `<a>`，并保留标题滚动偏移，避免跳转后被 Header 遮挡。
- 折叠目录优先使用原生 `<details>/<summary>`，无 JavaScript 也可操作。
- 推荐卡和上一篇 / 下一篇保持清晰的键盘焦点，点击目标不互相嵌套。
- 封面应有来自文章语义的替代文本；纯装饰回退图使用合适的空替代文本或隐藏语义。
- `prefers-reduced-motion: reduce` 下不执行位移动画；正文可见性不依赖动画状态。
- 页面禁用 JavaScript 后，封面、标题、正文、目录链接、推荐文章和前后篇导航仍完整可用。

## 实现边界

预计复用或调整以下职责，不在本设计阶段确定逐行实现：

- `src/layouts/ArticleLayout.astro`：文章顶部、正文、双栏和页尾顺序。
- `src/components/ArticleReadingNavigation.astro`：桌面目录与移动折叠目录。
- 新的轻量文章推荐组件：只负责渲染已计算结果。
- `src/lib/blog-directory.ts` 或相邻纯模块：推荐排序函数。
- `src/styles/interior.css` 及现有正文样式：文章面板、右栏和响应式表现。
- 相邻 Node 测试：推荐排序、排除当前文章、补齐和数量边界。

不修改 Content Collections schema、文章路由、RSS、站点正式 URL、全站音乐播放器或项目详情布局；不引入 React、Vue、Tailwind 或新的运行时依赖。

## 验收

自动验证：

```bash
node scripts/verify.mjs
```

浏览器验证至少覆盖：

- 一篇短文章和一篇最长文章。
- `1440×1000`、`1024×768`、`390×844`。
- 浅色与深色主题。
- 正常动效与 Reduced Motion。
- 启用和禁用 JavaScript。
- 桌面右栏推荐 3 条、吸顶目录、目录锚点跳转。
- 移动端折叠目录、正文后前后篇导航和页尾 2 条推荐。
- 外部直达文章后“返回博客”仍到达稳定目录页。
- 无正文永久透明、无横向溢出、无控制台错误。

完成标准：所有公开文章都能完整加载和阅读；页面结构与本文确认的桌面、移动顺序一致；推荐结果符合可测试规则；参考站只作为通用层级研究，不产生代码或素材复制。
