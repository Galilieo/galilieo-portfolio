## 参考来源与许可边界

XingHuiSama 仓库当前使用 Next.js、React、Tailwind、Framer Motion 和 Three.js，并采用 CC BY-NC 4.0。以下截图于 2026-07-12 从线上页面截取，仅用于设计研究和内部对照，不属于 Galilieo MIT 授权素材，也不得从 `src/` 导入。

允许提炼：

- 页面背景、主题遮罩和玻璃内容层的分离方式。
- 项目等权卡片矩阵、文章矩阵和中央时间线的布局原则。
- 日夜主题独立调色、卡片透明度、边框、阴影和响应式密度。
- 关于页的身份锚点、正文容器和成长叙事层级。

不得复制：

- 上游图片、头像、动漫素材、字体组合、精确色值、Tailwind 类和 React 组件。
- 搜索、播放器、评论、弹幕、宠物、悬浮工具、卡片/时间线切换等当前没有真实需求的功能。
- 虚构项目、文章、统计、活动记录或个人事实。

## 共用内页外壳

内页延续首页的雾蓝双主题玻璃系统，但内容面板比首页更实、更适合阅读。

- Header、Footer、导航和主题切换继续复用现有组件。
- 内页最大内容宽度约 `1120–1180px`，比首页 `1220px` 略窄。
- 内页与首页共用压缩后的全屏插画背景，但使用独立主题遮罩；文章/项目正文区域继续使用更实的阅读面板。
- 项目与博客列表通过卡片媒体区展示场景图；全页背景、主题遮罩、卡片图片和内容层保持解耦，未来替换素材不得重排页面结构。
- 用户提供素材仅按个人网站用途接入，公开再分发或商业化前必须重新核对原作者和许可。
- 桌面玻璃模糊约 `14–18px`；移动端关闭或降低大面积 `backdrop-filter`。
- 主卡圆角约 `22–24px`，内部媒体和标签使用更小圆角。

## 项目页

### 已确认方向

所有项目保持等权，不再通过 Heart Island 大卡和其他小卡区分尺寸。

桌面使用等尺寸项目矩阵：

- 宽桌面优先三列，三个真实项目在同一行形成完整矩阵。
- 中等视口降为两列；卡片尺寸和内部结构保持一致，第三张卡不拉伸成全宽。
- 移动端单列，不采用参考站在窄屏上的双列压缩。

每张卡使用同一结构：

1. 项目编号、类型和状态。
2. 项目名称与真实简介。
3. 角色或项目属性。
4. 最多 3–4 个技术标签。
5. 明确的详情入口；整卡可点击但不得嵌套交互。

视觉处理：

- 参考上游项目卡的等权玻璃面板、轻环境光和标签节奏。
- Heart Island 可以使用小范围受保护线稿作为卡内识别，不改变 SVG path、比例或水纹布局，也不让卡片尺寸变大。
- 其他项目没有正式 `cover` 时使用编号、项目类型和双主题 CSS 档案图形，不借用第三方图片。
- 当前只有三个项目，不加入搜索和筛选。

## 博客页

### 已确认方向

博客沿用等权文章卡的编辑式矩阵，并在内容增长后增加 Category 主目录，不再使用最新文章大卡或 Tag 筛选。

- 全部公开文章按 `category` 分组；分类按公开文章数量降序，数量相同时按 `zh-CN` 名称排序，组内按发布日期降序。
- 宽桌面使用左侧吸顶 Category 目录和右侧两列文章卡，分类链接使用真实 hash，不隐藏其他文章。
- 平板和移动端隐藏独立 Category 目录，直接显示按分类分组的文章流，避免重复信息挤占首屏。
- 桌面文章卡保持两列，保证中文长标题和摘要有足够宽度。
- 视口足够宽时仍保持两列，不为了填满页面改成过窄的四列。
- 移动端单列；不照搬参考站 390px 下的双列文章卡。

每张文章卡显示：

- 日期和分类。
- 标题和两行以内摘要。
- 2–3 个主要 Tag；Tag 只作为元信息，不承担筛选导航。
- 有 `cover` 时使用真实封面；没有时使用分类和 CSS 视觉占位。

`/blog/` 与 `/notes/` 复用同一 Category 数据和目录组件。仍不加入全文搜索、标签云、卡片/时间线切换或 Pagefind。

## 归档页

### 已确认方向

归档参考上游时间线视图的中央主轴链路：

- 桌面使用中央垂直主轴，文章卡按时间顺序左右交替。
- 主轴节点、年份/月分组和日期共同形成时间流。
- 当前只有四篇文章，卡片间距保持紧凑，不复制上游数十篇内容造成的超长空白。
- 卡片不使用封面，保持日期、分类、标题和短摘要的轻量信息结构。
- 背景氛围主要集中在页面上半部；正文时间线区域使用更深、更稳定的环境底色，保证主轴可读。

移动端：

- 主轴移到左侧，所有卡片排列在右侧。
- 日期和分类放在标题上方。
- 不依赖左右交替，不产生横向滚动。

不加入搜索、标签筛选、矩阵/时间线切换或客户端状态。

## 关于页

关于页保留此前确认的“个人档案”方向：

- 顶部使用共用插画背景与独立遮罩形成横幅，不复制上游封面。
- 使用站点所有者指定的人物头像，与 Header 和 favicon 保持一致。
- 直接展示 `Galilieo Atlas` 和“AI 应用开发者 / 软件工程学生”。
- 桌面采用约 `7:5` 的真实介绍与事实摘要布局；移动端自然单列。
- 不使用标签切换，核心内容全部服务端输出。

内容顺序：

1. Identity：身份、方向、当前状态。
2. Narrative：现有真实个人介绍。
3. Facts：当前身份、主要方向、当前项目、近期关注。
4. Timeline：软件工程学习、Heart Island 持续开发、AI 产品开发实习。使用一条连续纵向主轴、三个阶段节点和横向档案记录行，不再使用三张并排阶段卡；移动端收窄为左轴单列。当前已确认时间分别为 `2024.09 — 至今`、`2026.04 — 至今`、`2026.06 — 至今`；日期作为主元信息，`01 / Learning`、`02 / Building`、`03 / Practice` 作为辅助阶段标识。
5. Contact：GitHub、邮件和 Heart Island 真实入口。

不加入评论、GitHub 活跃度、动态 API 或社交主页功能。

## 项目与文章详情

列表页和详情页共用内页背景，但正文使用更实的阅读面板：

- 项目详情突出项目名称、角色、状态和技术栈。
- 文章正文保持舒适阅读宽度，继续支持代码块、表格和前后篇导航。
- 文章详情从 Markdown H2/H3 生成正文目录；宽桌面使用右侧吸顶目录，平板和移动端在文章头部只保留“本文目录”折叠面板。
- 没有正式封面时不渲染巨大的空媒体区。
- 只增加正文目录，不加入作者、热门文章和评论侧栏。
- 无 JavaScript 时正文、分类分组与章节链接必须完整可读；移动端不依赖双视图脚本。

## 响应式与交互边界

- 代表视口：1440×1000、1024×768、390×844。
- 日间和夜间都是正式状态，不简单反色。
- 移动端卡片单列优先，减少模糊和外围装饰。
- Hover 只做约 `2px` 以内位移、边框和阴影变化；触摸设备不依赖 Hover。
- Reduced Motion 停止持续动画和位移。
- Reveal、移动菜单和 Astro 页面切换继续使用现有生命周期和无 JavaScript 回退。

## 实施顺序

审批后按以下顺序执行：

1. 共用内页背景、遮罩、宽度和玻璃 token。
2. 项目等权矩阵与博客 `2 × 2` 文章矩阵。
3. 归档中央主轴链路与移动左轴降级。
4. 关于页身份档案、真实叙事和成长时间线。
5. 项目/文章详情阅读面板一致性。
6. 自动验证和多视口、双主题、Reduced Motion、无 JavaScript 浏览器验收。

## 实施与验收记录

2026-07-12 已按本文方案完成生产实现：

- 项目页为三个等尺寸卡片；1440px 三列、1024px 两列、390px 单列。
- 博客页和 `/notes/` 使用同一 Category 目录与分组；组内桌面两列、移动端单列。
- 归档桌面为中央主轴、左右交替、短连接线和 End 终点；移动端为左轴单列。
- 关于页包含身份横幅、约 `7:5` 的叙事/自适应事实卡、三段真实成长记录和联系区。
- 项目与文章详情使用更实的阅读面板，正文宽度和静态阅读能力保持不变。
- 文章详情使用 H2/H3 正文目录；宽桌面右侧吸顶，平板和移动端收敛为文章头部的单一本文目录。
- 内页共用压缩插画背景，列表卡媒体区使用 Category/项目场景图，阅读面板维持独立日夜透明度；移动端关闭大面积模糊并提高底色不透明度。

## 不变量

- 不修改 Heart Island 受保护 SVG path、主体比例、水纹布局和既定动效语言。
- 不虚构内容、封面、统计、经历或数据。
- 不改变 Content Collections、路由、RSS、Sitemap、Nginx 和生产配置。
- 不改变首页两行仪表盘结构。
- 不引入 React、Vue、Tailwind、Framer Motion、Three.js 或新的视觉运行时。
